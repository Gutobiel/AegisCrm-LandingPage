# PROJECT — Aegis CRM Landing Page Redesign

## 1. Project Goal
Transform the Aegis CRM landing page from a generic "SaaS template" into a unique, highly humanized, and authentic page that strictly follows `.cursorrules` (typography focus, no symmetric grids, refined colors, no AI jargon).

## 2. Requirements (R1-R5)
- **R1 - Brainstorm Document**: A comprehensive strategy referencing `.cursorrules`. *(Partially Completed)*
- **R2 - Authentic Copy**: Real-world sales copy. Zero jargon. Testimonials with specific details. *(In Progress)*
- **R3 - Anti-Generic Layout**: Asymmetry, bento grids, varied whitespace. Break the 3-column curse. *(In Progress)*
- **R4 - Real Visual Assets**: Product screenshots and photographic avatars. *(Completed)*
- **R5 - Organic Micro-interactions**: Refined blue/black/white/gray tones. Subtle, elegant animations replacing templates like AOS/confetti. *(In Progress)*

## 3. Milestones
- **M1: Strategic Review & Brainstorm Expansion**
  - Synthesize the current state.
  - Deliver a definitive brainstorm artifact.
- **M2: Advanced Code Redesign (HTML & CSS)**
  - Overhaul typography rules to enforce `.cursorrules`.
  - Refine color hexes (no Tailwind defaults).
  - Implement full Bento UI components with organic padding.
- **M3: Organic Interactivity (JS)**
  - Refactor micro-interactions.
  - Implement custom easing and staggered animations.

## 4. Subagent Roster
- `copy_expert`: Focuses on eliminating corporate/AI jargon and ensuring tone is authentic Brazilian Portuguese.
- `ui_designer`: Focuses on `.cursorrules` enforcement (typography, color refinement, layout asymmetry).
- `frontend_engineer`: Executes complex HTML/CSS/JS structural changes based on the designer's specifications.
